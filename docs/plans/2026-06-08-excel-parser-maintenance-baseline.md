## Context

This repository contains a compact Python 2-era `xlrd` helper in `parse.py`.
The parser exposes callback-based row processing and type conversion, but the
module currently uses Python 2-only exception syntax and imports `xlrd`
unconditionally. That makes local verification fail on modern Python even for
conversion behavior that does not need a real workbook.

## Goals

- Keep the existing callback API and conversion constants intact.
- Make `parse.py` importable and testable under Python 3 without requiring
  `xlrd` for fake-workbook tests.
- Preserve Python 2.7 compatibility for the parser module.
- Add unit tests for conversion behavior, header skipping, missing cells, parse
  completion, and exception callbacks.
- Document the `xlrd` 2.x `.xls` dependency boundary and safe fixture
  expectations.

## Scope Boundaries

- Do not migrate the project to a broad spreadsheet framework.
- Do not claim date conversion support.
- Do not add real or private spreadsheet fixtures.
- Do not change the row callback, parse-done callback, or exception callback
  shape.

## Implementation Units

### U1: Parser compatibility

Files: `parse.py`, `requirements.txt`

Approach: Keep `xlrd` constants available through a local fallback when the
dependency is absent, make exception syntax Python 2/3-compatible, avoid a
mutable default argument, open workbooks on demand, release workbook resources,
and raise a clear dependency error when processing a real workbook without
`xlrd`. Track the maintained `xlrd` 2.x line and document that this baseline
covers `.xls` workbooks, not modern `.xlsx` files.

Verification: `python3 -m py_compile parse.py` passes; `python2 -m py_compile
parse.py` runs when Python 2 is available.

### U2: Offline parser tests

Files: `tests/test_parse.py`

Approach: Use `unittest` with a fake `xlrd` workbook object so tests can cover
row processing without a binary workbook fixture or third-party dependency.

Verification: `python3 -m unittest discover -s tests -p "test*.py"` passes.

### U3: Maintenance guardrails

Files: `Makefile`, `scripts/check-baseline.sh`, `README.md`, `VISION.md`,
`CHANGES.md`, `.gitignore`

Approach: Add a repeatable baseline command, document the `.xls` dependency
boundary and test path, and ignore generated bytecode.

Verification: `scripts/check-baseline.sh`, `make check`, and `git diff --check`
pass.
